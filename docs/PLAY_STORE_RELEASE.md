# 플리로그 · 구글 플레이스토어 배포 가이드

이 문서는 **플리로그(plilog)** 앱을 구글 플레이스토어 심사 제출까지 진행하기 위한
**처음부터 끝까지의 절차**를 담고 있습니다. 위에서부터 순서대로 따라 하면 됩니다.

> 이 앱은 Expo + EAS 기반입니다. 안드로이드 네이티브 폴더(`android/`)를 직접 다루지 않고
> **EAS 클라우드 빌드**로 출시용 `.aab`(Android App Bundle)를 만듭니다.

---

## 0. 한눈에 보는 전체 흐름

```
계정 준비 → 출시 전 설정 점검 → 프로덕션 빌드(.aab) → Play Console 앱 생성
→ 스토어 등록정보/이미지 입력 → 콘텐츠 등급·데이터 보안·개인정보처리방침
→ 내부 테스트 업로드 → 프로덕션 트랙 제출 → 심사 → 출시
```

처음 출시는 보통 **반나절~2일**(구글 심사 대기 포함) 정도 걸립니다.

## 2. 출시 전 설정 점검 (코드/설정)

배포 전에 아래 항목을 확인합니다. 대부분 이미 맞춰져 있습니다.

### 2-1. `app.json` 확인

현재 값(점검용):

- `expo.name`: **"플리로그"** (앱 이름)
- `expo.version`: **"1.0.0"** ← 사용자에게 보이는 버전. 업데이트마다 올림 (예: 1.0.1)
- `expo.android.package`: **"com.playlist.app"** ← **한 번 출시하면 절대 변경 불가**. 지금 확정.
- 권한: `RECORD_AUDIO`, `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_MEDIA_PLAYBACK`, `WAKE_LOCK` 등

> `versionCode`(안드로이드 내부 빌드 번호)는 `eas.json`의 `production` 프로필에
> `"autoIncrement": true`가 설정되어 있어 **빌드할 때마다 자동으로 1씩 증가**합니다.
> 직접 관리할 필요 없습니다.

### 2-2. 분석(Analytics) 키 설정 — 선택

`lib/analytics.ts` 상단의 `POSTHOG_API_KEY`를 채우면 사용자 행동 분석이 켜집니다.
- **키를 채운 경우** → 8번 "데이터 보안"에서 **데이터 수집 항목을 반드시 선언**해야 합니다.
- **비워둔 경우** → 아무 데이터도 수집하지 않으므로 별도 선언 불필요.

(분석 설정 방법은 `lib/analytics.ts` 파일 주석 + 본 문서 8-2 참고)

### 2-3. 빌드가 깨지지 않는지 사전 점검

```bash
npm test            # 유닛/컴포넌트 테스트 (전부 통과해야 함)
npx tsc --noEmit    # 타입 점검 (expo-audio 관련 기존 경고 2건은 무시 가능)
```

## 3. 프로덕션 빌드 만들기 (.aab)

플레이스토어에 올리는 정식 형식은 **AAB(Android App Bundle)** 입니다.
(개발/내부 배포용 APK와 다릅니다.)

```bash
eas build --platform android --profile production
```

- 처음 실행하면 **앱 서명 키(keystore)를 EAS가 자동 생성/보관**할지 묻습니다 → **Yes** 권장.
  (직접 관리하려면 4번 참고. 대부분은 EAS 자동 관리가 편하고 안전합니다.)
- 빌드는 클라우드에서 진행되며 완료되면 **`.aab` 다운로드 링크**가 나옵니다.
- 빌드 상태는 https://expo.dev 의 프로젝트 → Builds 에서도 확인 가능합니다.

빌드가 끝나면 `.aab` 파일을 받아둡니다(5번 또는 9번에서 사용).

## 4. 앱 서명 (Google Play 앱 서명)

- 구글은 **Play 앱 서명**(Google이 최종 서명 키를 보관)을 기본으로 사용합니다.
- EAS가 만든 **업로드 키**로 서명한 `.aab`를 올리면, 구글이 받아서 최종 서명합니다.
- **권장 구성**: 서명 키를 EAS에 맡기고(`eas build`가 자동 처리), Play 앱 서명도 그대로 사용.
- 키는 분실하면 **앱 업데이트가 불가능**해지므로, EAS에 보관된 키는 백업해 두면 좋습니다:

```bash
eas credentials      # Android → Keystore 확인/다운로드 가능
```

## 5. Play Console에서 앱 생성

1. https://play.google.com/console 접속 → **앱 만들기(Create app)**
2. 입력:
   - **앱 이름**: `플리로그`
   - **기본 언어**: 한국어
   - **앱 / 게임**: 앱
   - **무료 / 유료**: 무료

## 8. 데이터 보안 (Data safety) — ⚠️ 중요

좌측 **앱 콘텐츠 → 데이터 보안** 설문. 실제 동작과 정확히 일치해야 심사를 통과합니다.

### 8-1. 녹음(마이크)에 대해

- 이 앱의 녹음 파일은 **기기 내부에만 저장**되며 서버로 전송하지 않습니다.
- 구글 기준에서 "수집(collection)"은 **데이터가 기기 밖으로 전송될 때**를 뜻합니다.
- 따라서 **오디오/음성을 "수집" 항목으로 선언하지 않습니다.**
  (마이크 권한은 사용하지만, 데이터는 외부로 나가지 않음)

### 8-2. 분석(Analytics)에 대해 — `POSTHOG_API_KEY`를 채운 경우에만

분석을 켰다면 아래처럼 선언하세요:
- **데이터를 수집합니까? → 예**
- 데이터 유형: **앱 활동 → 앱 상호작용**(화면 조회/버튼 클릭 등)
- 목적: **분석(Analytics)**
- **제3자와 공유? → 아니오** (분석 도구 사용은 처리 위탁이며, 광고 목적 공유 아님)
- 사용자와 연결되는 식별자: **익명 기기 식별자만 사용**(이름·이메일 등 개인정보 없음)

> 분석 키를 **비워두면** 데이터를 전혀 수집하지 않으므로
> "데이터를 수집/공유하지 않음"으로 선언하면 됩니다.

## 12. 빌드 제출 방법 (두 가지 중 택1)

### 방법 A — 명령어로 자동 제출 (권장)

```bash
eas submit --platform android --profile production --latest
```
- 처음에는 **Google 서비스 계정 키(JSON)** 연결을 요구합니다.
  Play Console → 설정 → API 액세스 → 서비스 계정 생성 후 JSON 키를 발급해
  `eas.json`의 submit 설정 또는 명령 안내에 따라 연결합니다.
- 연결되면 최신 빌드를 지정 트랙으로 자동 업로드합니다.

### 방법 B — 수동 업로드

- 3번에서 받은 `.aab` 파일을 Play Console의 해당 트랙에서 직접 업로드.
- 처음 한 번은 이 방법이 더 단순합니다(서비스 계정 설정 불필요).

## 14. 이후 업데이트 배포

코드 수정 후:

1. `app.json`의 `expo.version`을 올림 (예: `1.0.0` → `1.0.1`)
   - `versionCode`는 `eas.json`에서 자동 증가하므로 건드리지 않아도 됨.
2. 빌드:
   ```bash
   eas build --platform android --profile production
   ```
3. 제출:
   ```bash
   eas submit --platform android --profile production --latest
   ```
4. Play Console에서 새 버전 출시 노트 작성 후 출시.

| "App Bundle을 업로드할 수 없음 / 서명 오류" | APK가 아니라 **AAB**인지, EAS 서명 키가 일관적인지 확인 |
| "package 이름이 이미 사용 중" | `com.playlist.app`이 다른 앱과 충돌 → app.json에서 고유하게 변경(첫 출시 전에만 가능) |
| "데이터 보안 설문과 권한 불일치" | 마이크 권한을 쓰면 데이터 보안/개인정보처리방침을 반드시 작성 |
| 프로덕션 트랙이 안 열림 | 신규 개인 계정의 **비공개 테스트 요건**(11번) 미충족 가능 |
| `versionCode` 중복 | 같은 빌드 재업로드 시 발생 → 새로 `eas build` 실행해 번호 증가 |
| 심사 반려: 개인정보처리방침 | 공개 URL이 실제 접속 가능한지, 마이크 사용 설명이 포함됐는지 확인 |
